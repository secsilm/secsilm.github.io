PyCon China 2020 参会感受

2020 年 11 月 30 日

目录

  1. 大会组织
  2. 讲师
  3. 总结
  4. END

上周末在北京参加了 PyCon China 2020，这是第一次参加 PyCon China，有一些简单感受，在此简要记录下。

当天参会完之后，我简单在即刻上发了一条动态吐槽：

即刻动态

下面我展开来说下。

大会组织

首先必须要说的是和上海那边的同步。这次大会由于疫情原因取消了现场演讲，采取的是讲师提前录播然后现场播放的方式。上午是北京上海深圳三地同时播放三位讲师的录播视频，然后会现场连线讲师进行 QA。注意是三地同时播放，怎么个同时法？就是北京深圳都转播上海的视频，北京深圳屏幕共享看上海的视频。

这时体验就比较差了。不知道是否是收音的问题，上海那边主持人说话根本听不清。最开始的时候北京这边一点声音都没有，看到有人和上海那边反馈，但是最后也没啥效果。而且就好像没有排练一样，主持人经常会和台下的工作人员交流，给人一种不连续的感觉。而且无论是穿着还是姿态还是说话，都给人一种不正式的感觉，过于随意，作为带有 China 的这种国字头的大会来说，个人感觉实在是不应该。当然这种情况也存在于北京。

不知道会前是否有进行彩排和联调测试，在现场发生这样的问题实在是太不应该了。如果有测试这一定能被测出来。还发生了几次视频放错的情况，实在是不应该。

关于签到和引导。下午的时候我在外面透气，两次看到有人刚来参会，但是却找不到故宫厅（会议举行的地方）在哪，都走过了，而且前台明明有志愿者在。缺乏主动引导。

但是大家都是志愿来做这件事的，还是老罗那句话：又不是不能用。但是有有一句话：既然做了，就要做好。

讲师

上面说的只是会议组织上的问题，其实问题不算严重，毕竟来参会是来听讲师们的精彩议题的。但是议题却不怎么精彩。

首先说说上午的三个议题。PSF VP 马琳主要是讲了一些不痛不痒的关于她自己的经历、PSF 和 Python Africa 之类的。但是中间她说的 Python Africa 还是什么的会议组织，要让参会者感觉到 welcome 和 celebrated，我很赞同。最起码这次会议中我没有感到 welcome 和 celebrated。

第二个是微软 Visual Studio Code Python 首席产品经理陆榕，title 听着很高。其实演讲上没什么大问题，就是一个自家产品的广告，但是又太浅，就像马保国老师说的传统功夫讲究点到为止。很明显她也没有掌握基本的 AI 常识，一些用词不太恰当，不过也不能要求太高。总之就是很浅，让新手听着不知所云，让老手觉得索然无味。

第三个老哥一听就是真正的硬核技术党，Fast API 作者，而且在 spacy 做了 prodigy 这样漂亮的产品。可惜的是我后来有点事缺席了，回头再来看一看。

下午的可就真的是参差不齐了，而且 CI 的占绝大多数，我就不明说是哪些讲师了。这些讲师可能本身水平确实很高，但是当他讲出来，讲给别人听的时候，真是让人昏昏欲睡，和机器人在那念稿一样。刚才说微软那个太浅，现在这些则是太深，而且还没有适当地引入，没有过渡，直接就 dive 到最深的沟沟里。不是做这方面的或者水平不及的观众，可能就是不明觉厉，但是这没有用，观众没有得到有效信息。而且一些演讲缺乏条理性。

其实很多的技术人应该都有这个问题，这里也不是怼讲师们，毕竟大家也是花了自己的宝贵时间来做分享的。也不是说我能做得很好，我只是评论一下，那些 “You can you up, no can no bb” 的话就不要说了，我也不想争论。知道洞的存在，你才能把这个洞补上。

总结

大会组织和讲师演讲上都还有很大可改进的地方，希望 PyCon China 在下个十年，能做得更好，更有影响力。
